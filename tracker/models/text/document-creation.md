# Создание документов

Практическая инструкция по запуску генерации документов через режим `document=true` в трекере.

## Как отправлять запрос

- Ссылка: `https://api.pxsto.re/main/puzzlebot-tracker`
- Метод: `POST`
- Вид запроса в PuzzleBot: `Сформированный`
- Формат тела: `application/json` или те же ключи в no-code параметрах.

Для новых запросов используйте ключ модели с подчёркиванием, например `gpt_5`, а не `gpt-5`.

## Пример запроса с комментариями

```jsonc
{
  "bot": "{{BOT_USERNAME_TEXT}}", // обязательно: username бота в PuzzleBot.
  "token": "[Ваш API-токен]", // обязательно: API-токен бота для доступа к трекеру.
  "user": "{{USER_ID_TEXT}}", // обязательно: ID пользователя или сессии.
  "model": "gpt_5", // обязательно: текстовая модель, которая подготовит содержимое документа.
  "document": true, // обязательно: включает режим генерации документа.
  "format": "pdf", // обязательно: формат файла, например `pdf`, `docx` или `odt`.
  "template": "modern", // рекомендуется: шаблон документа; используйте `custom` для своего шаблона.
  "style": "modern", // необязательно: старый параметр оформления, оставлен для совместимости.
  "custom_template": "Договор оказания услуг: ...", // необязательно: нужен только при `template=custom`.
  "custom_font": "Montserrat", // необязательно: шрифт документа, если он доступен в настройках.
  "role": "Ты опытный юрист", // рекомендуется: роль, которая задаёт экспертизу и стиль документа.
  "prompt": "Сформируй договор на основе данных: {{client}}", // обязательно: задача и данные для документа.
  "send_answer": true, // необязательно: отправить готовый файл пользователю; по умолчанию `true`.
  "chat": "-1001234567890", // необязательно: ID чата или канала для отправки результата.
  "topic": 12 // необязательно: ID топика форума, если отправляете в топик.
}
```

## Параметры

| Параметр | Тип | Обязательный | Описание |
| --- | --- | --- | --- |
| `bot` | string | Да | Username бота. В PuzzleBot обычно используется `{{BOT_USERNAME_TEXT}}`. |
| `token` | string | Да | API-токен бота для авторизации запроса. |
| `user` | string | Да | ID пользователя или сессии. В PuzzleBot обычно используется `{{USER_ID_TEXT}}`. |
| `model` | string | Да | Точный ключ вызываемой модели: `gpt_5`. |
| `document` | boolean | Да | Должно быть `true`, чтобы трекер запустил генерацию документа. |
| `format` | string | Да | Формат готового файла: `pdf`, `docx` или `odt`. |
| `prompt` | string | Да | Задача и исходные данные для документа. |
| `role` | string | Рекомендуется | Роль модели: юрист, HR, методист, аналитик и т. д. |
| `template` | string | Нет | Предпочтительный параметр шаблона: `classic`, `modern` или `custom`. |
| `style` | string | Нет | Старый параметр оформления. Работает для совместимости, но лучше использовать `template`. |
| `custom_template` | string | Условно | Нужен только при `template=custom`. Алиас для совместимости: `customTemplate`. |
| `custom_font` | string | Нет | Шрифт документа. Алиас для совместимости: `customFont`. |
| `send_answer` | boolean | Нет | `true` отправляет результат пользователю. `false` не отправляет сообщение и сохраняет результат в `{{tracker_answer}}`. По умолчанию `true`. |
| `chat` | string/number | Нет | ID чата или канала для отправки результата. |
| `topic` | number | Нет | ID топика форума. |

## Ответ

```json
{
  "answer": "Ссылка на готовый документ."
}
```

Если одновременно переданы `template` и `style`, приоритет у `template`.
