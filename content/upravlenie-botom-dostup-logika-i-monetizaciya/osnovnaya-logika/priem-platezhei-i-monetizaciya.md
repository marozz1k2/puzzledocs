# Прием платежей и монетизация

Когда у пользователя заканчиваются бесплатные попытки, бот должен предложить ему купить новый пакет. В конструкторе PuzzleBot этот процесс можно настроить: как только проходит оплата, переменная баланса (например, наша `req_exp`) мгновенно увеличивается на купленное количество запросов.

#### Шаг 1. Подключение платежной системы

Прежде чем создавать товары в боте, нужно подключить кошелек, куда будут приходить деньги (ЮKassa, Robokassa, Stripe и др.).

1. В личном кабинете PuzzleBot перейдите в раздел «Настройки» -> «Платежные системы».
2. Выберите подходящий сервис и следуйте инструкции по подключению (вам нужно будет получить токен от платежной системы).
3. После успешного подключения в конструкторе появится возможность создавать логику приема платежей.

#### Шаг 2. Создание команды для покупки

Теперь создадим команду, которую увидит пользователь, когда захочет пополнить баланс.

1. В Конструкторе создайте новую команду (например, `buy_100_req`).
2. Добавьте текстовый блок с описанием: _«Пакет: 100 AI-запросов. Стоимость: 500 ₽»_.
3. Добавьте Инлайн-кнопку и выберите действие «Платеж». В нем укажите:
   * Наименование платежа: 100 запросов к ИИ.
   * Стоимость: 500.
   * Валюта: RUB.
   * Команда при успешной оплате: Создайте и выберите техническую команду (например, `pay_success_100`).

<figure><img src="../../.gitbook/assets/image (259).png" alt=""><figcaption></figcaption></figure>

#### Шаг 3. Настройка логики после оплаты

Это самый важный момент — нам нужно, чтобы цифры на балансе пользователя изменились сразу после списания денег с его карты.

1. Откройте созданную команду `pay_success_100`.
2. В разделе «Действия» выберите «Изменить переменную».
3. Выберите вашу переменную баланса (`req_exp`) и укажите выражение: `+100`.
4. (Опционально) Добавьте действие «Изменить категории» -> «Удалить» -> `gpt_mute`. Это нужно, если пользователь был заблокирован из-за нулевого баланса — оплата автоматически снимет с него все ограничения.

#### Шаг 4. Где разместить кнопку оплаты?

Чтобы пользователь не искал, как вам заплатить, добавьте кнопку пополнения в два места:

* **В Личном кабинете:** Создайте команду «Профиль», где через переменную выводится остаток (`Ваш баланс: {req_exp}`), и прикрепите кнопку «Пополнить».
* **В сообщении об ошибке:** Помните нашу команду `TextModels_error` (когда запросы кончились)? Замените сухой текст «Запросы кончились» на предложение: _«У вас закончились лимиты. Купите пакет из 100 запросов, чтобы продолжить работу»_ и добавьте кнопку оплаты.
