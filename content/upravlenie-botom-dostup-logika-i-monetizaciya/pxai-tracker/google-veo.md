# Google Veo

Функционал предназначен для продвинутых пользователей, которые хотят получить полный контроль над созданием кинематографичных видеороликов через Google Veo 3.1.

**Для чего это нужно?**

Хотя Veo 3.1 поддерживается в Puzzle AI по умолчанию, дополнительные возможности Tracker позволят:

* Создавать кастомные мини-приложения с выбором качества (`fast`/`quality`).
* Гибко управлять параметрами (ориентация, длительность).
* Использовать уникальные функции Veo: "оживление" одного фото или создание плавного перехода между двумя кадрами.
* Встраивать Veo 3.1 в различные сценарии.

#### Настройка запроса

В любой команде вашего бота (например "`veo_tracker`") добавьте действие «Отправить запрос» и настройте его:

* Ссылка: `https://api.pxsto.re/main/puzzlebot-tracker`
* Тип запроса: `POST`
* Вид запроса: `Сформированный`

### Практические примеры

Вы можете использовать Puzzle AI Tracker в связке с Veo 3 для разработки собственных телеграм-ботов и мини-приложений.

#### Кейс 1. Делаем плавный переход между 2 кадрами

Это уникальная функция Veo. Вы можете создать бота, который анимирует переход между двумя изображениями.

**Задача:** Пользователь загружает кадр "До" и кадр "После". Бот создает плавный видеопереход без необходимости вводить текстовый запрос.

**Решение:**

1. Создайте команду, например, `/transition`.
2. В ней сделайте две «Формы ввода» для картинок:
   * Форма 1 (Старт):
     * Текст: `Загрузите НАЧАЛЬНЫЙ кадр.`
     * Тип ввода: `Отправка сообщения`
     * Маска ввода: `Изображение`
     * Переменная: `{{frame_start}}`
   * Форма 2 (Финал):
     * Текст: `Загрузите КОНЕЧНЫЙ кадр.`
     * Тип ввода: `Отправка сообщения`
     * Маска ввода: `Изображение`
     * Переменная: `{{frame_end}}`

<figure><img src="../../.gitbook/assets/image (227).png" alt=""><figcaption></figcaption></figure>

3. Сразу после нее — действие «Отправить запрос» (Tracker) со статическим промптом:

| **Ключ**                   | **Значение**                                                                                              | **Зачем?**                                                   |
| -------------------------- | --------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| `images`                   | `{{frame_start}},{{frame_end}}`                                                                           | Передаем ОБА кадра.                                          |
| `prompt`                   | `Create a smooth, seamless video transition that logically connects the first image to the second image.` | Ваш «мастер-промпт» для перехода. Пользователь его не видит. |
| `model`                    | `veo`                                                                                                     | Для такой задачи нужно лучшее качество.                      |
| (и `user`, `bot`, `token`) |                                                                                                           |                                                              |

**Результат:** Ваш пользователь получает готовый ролик, показывающий трансформацию "до/после", просто загрузив два фото.

#### Кейс 2. Бот для риелторов

**Задача:** Риелтор загружает 1 фото комнаты. Бот должен создать не просто "оживление", а дорогой кинематографичный облет этой комнаты.

**Решение:**

1. Создайте команду, например, `/room_tour`.
2. В ней сделайте одну «Форму ввода»:
   * Текст: `Загрузите 1 фото вашей комнаты, и я создам для нее кинематографичный облет.`
   * Тип ввода: `Отправка сообщения`
   * Маска ввода: `Изображение`
   * Переменная: `{{room_photo}}`

<figure><img src="../../.gitbook/assets/image (64).png" alt=""><figcaption></figcaption></figure>

3. В действии «Отправить запрос» (Tracker) используйте профессиональный промпт:

| **Ключ**                   | **Значение**                                                                                                                                                    | **Зачем?**                                    |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| `images`                   | `{{room_photo}}`                                                                                                                                                | Берем 1 фото комнаты.                         |
| `prompt`                   | `Animate this photo. Cinematic drone shot, slowly flying backwards and slightly upwards, revealing the whole room. Bright, spacious, luxury real estate style.` | Ваш «мастер-промпт» с командами для камеры.   |
| `model`                    | `veo`                                                                                                                                                           | Для такой задачи нужно лучшее качество.       |
| `orientation`              | `horizontal`                                                                                                                                                    | Для недвижимости нужен горизонтальный формат. |
| (и `user`, `bot`, `token`) |                                                                                                                                                                 |                                               |

**Результат:** Риелтор получает профессиональный ролик, который можно вставить в презентацию.
